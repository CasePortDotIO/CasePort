# CasePort B2B Homepage - Visual Richness Audit

## Key Visual Elements Missing from Injured Page

### 1. **Typography Sophistication**
- **Headline hierarchy**: Large, bold headlines with strategic color accents (cyan on key words)
- **Subtitle styling**: Smaller, lighter text that guides the eye
- **Label styling**: Small caps, uppercase labels with letter spacing (e.g., "FOR GROWTH-ORIENTED PERSONAL INJURY LAW FIRMS")
- **Font weights**: Mix of 400, 600, 700, 800 for visual hierarchy

### 2. **Spacing & Layout Rhythm**
- **Generous vertical spacing**: Large gaps between sections (80px-120px)
- **Consistent padding**: 40px-60px padding on sections
- **Breathing room**: Content doesn't feel cramped
- **Grid alignment**: 12-column grid with consistent gutters

### 3. **Visual Elements & Borders**
- **Subtle dividers**: Light borders between sections (not heavy)
- **Background colors**: Alternating light/dark sections for visual rhythm
- **Accent lines**: Thin cyan lines as visual separators
- **Cards with borders**: Subtle 1px borders on cards (not shadows)

### 4. **Micro-interactions & Animations**
- **Hover effects**: Buttons scale up, colors shift
- **Transitions**: Smooth 300ms transitions on all interactive elements
- **Scroll animations**: Elements fade in as you scroll
- **Button states**: Active/hover/disabled states are clear

### 5. **Color Layering**
- **Background gradients**: Subtle gradients on dark sections
- **Color overlays**: Semi-transparent overlays on images
- **Accent colors**: Cyan used strategically (not everywhere)
- **Text colors**: Multiple shades of gray for hierarchy

### 6. **Component Polish**
- **Buttons**: Rounded corners (8px), proper padding (12px 24px), clear hover states
- **Input fields**: Clean borders, focus states with glow
- **Cards**: Proper shadow or border, consistent padding
- **Icons**: Consistent sizing and weight

### 7. **Data Visualization**
- **Metric cards**: Large numbers with descriptive text below
- **Process flow**: Visual 6-step flow with connecting lines
- **Comparison tables**: Clear visual hierarchy, alternating row colors
- **Charts**: Simple, clean data visualization (not cluttered)

### 8. **Specific Design Patterns**
- **Hero section**: Asymmetric layout with copy on left, visual on right
- **Feature comparison**: Two-column "Without CasePort" vs "With CasePort"
- **Testimonials**: Quote marks, author names, company logos
- **CTAs**: Multiple CTAs with different styles (primary, secondary, tertiary)

## What to Apply to Injured Page

1. **Typography**: Add more font weight variety, use small caps for labels
2. **Spacing**: Increase vertical spacing between sections (currently too tight)
3. **Borders & Dividers**: Add subtle 1px borders instead of heavy shadows
4. **Micro-interactions**: Add hover effects, scroll animations
5. **Color layering**: Add subtle background colors to alternate sections
6. **Component polish**: Refine button styles, input focus states
7. **Data visualization**: Make metric cards more prominent
8. **Layout**: Use asymmetric layouts instead of centered grids

## Color Rhythm to Maintain
- Keep cyan (#0891B2) as primary accent
- Keep white backgrounds
- Keep dark text on light backgrounds
- Don't change the overall color scheme
